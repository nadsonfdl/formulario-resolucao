Guia para Gerar a Resolução

  • 1. Qual foi o problema ou solicitação do cliente?

    Detalhe o que foi relatado pelo cliente, o que ele esperava e qualquer informação relevante que ele forneceu. Isso inclui o histórico e contexto do problema.

  • 2. Quais abas do sistema foram acessadas?

    Explique quais abas ou menus dentro do sistema o agente precisou acessar para realizar o atendimento. Exemplo: "Prontuário > Histórico do paciente" ou "Configurações > Segurança".

  • 3. Quais configurações foram alteradas ou verificadas?

    Caso tenha sido necessário alterar configurações, cite as configurações acessadas ou modificadas no sistema, como opções de impressão, permissões de usuário, ou configurações de módulo.

  • 4. Alterações realizadas no banco de dados?

    Se houve necessidade de ajustes no banco de dados, descreva quais tabelas e colunas foram alteradas, e se foi utilizada alguma query para ajustar dados. Exemplo: "Tabela: unidadesconfig, Coluna: LIBERAR_ORCAMENTO_MANUAL, Query: UPDATE unidadesconfig SET LIBERAR_ORCAMENTO_MANUAL='S' WHERE ..."

  • 5. Qual a resolução técnica aplicada?

    Descreva as ações tomadas, como foi feito o ajuste técnico, e se foi necessário realizar alguma interação manual, como ajustes na interface ou mesmo no código.

Exemplo de Resolução:

QUESTÃO DO CLIENTE: O cliente relatou que ao tentar imprimir um orçamento gerado manualmente no sistema, o documento estava saindo em branco.

RESOLUÇÃO: Acesse o módulo "Prontuário", vá para "Pre-Op", e altere as configurações de impressão para garantir que todos os dados sejam exibidos corretamente. Após isso, verifique as permissões de acesso e habilite a função de "Gerar Orçamento Manual" no banco de dados, se necessário.

Resolução Gerada:

A resolução será gerada aqui.
